# Контроль сроков годности партий товаров

Веб-сервис для учета партий товаров, контроля сроков годности, формирования отчетов, импорта/экспорта XLSX и ежедневных email-уведомлений. Проект работает по адресу:

```text
https://kvasmix.ru/vr/sroki_godnosti/
```

Новая архитектура проекта:

```text
Сайт → PHP API → MariaDB
```

## Возможности

- Вкладка `Загрузка партии`:
  - ручное добавление партии;
  - скачивание шаблона XLSX;
  - импорт XLSX с колонками `Артикул`, `Количество`, `Срок годности до`;
- Вкладка `Реестр`:
  - список всех партий;
  - поиск по артикулу;
  - фильтры по статусу и остатку дней: просроченные, 15, 30 или 60 дней;
  - изменение статуса партии;
  - редактирование всех полей партии через окно редактирования;
  - удаление партии с сохранением действия в истории;
  - выгрузка полного списка или результата фильтрации в XLSX без партий со статусами `Реализована` и `Списана`.
- Вкладка `Настройки`:
  - добавление и удаление email-получателей;
  - отображение фиксированных критериев отправки: срок истек, осталось 15, 30 или 60 дней.
- Вкладка `История`:
  - история добавления, импорта, редактирования, изменения статусов и удаления партий в реестре.
- Цветовая индикация в реестре:
  - без индикации — более 60 дней;
  - желтый — до 60 дней;
  - оранжевый — до 30 дней;
  - красный — до 15 дней;
  - серый — просрочено.

## Требования к серверу Timeweb VPS

- Linux VPS с доступом по SSH.
- Nginx или Apache.
- PHP 8.0 или новее.
- PHP-расширения: `pdo_mysql`, `mbstring`, `json`.
- MariaDB 10.5+ или MySQL 8+.
- Настроенная отправка почты с сервера для функции PHP `mail()` или внешний MTA.
- Cron для ежедневного запуска проверки сроков.

## Установка MariaDB

Пример для Ubuntu/Debian:

```bash
sudo apt update
sudo apt install mariadb-server mariadb-client
sudo systemctl enable --now mariadb
sudo mysql_secure_installation
```

Проверьте, что MariaDB запущена:

```bash
systemctl status mariadb
```

## Создание базы данных и пользователя

Подключитесь к MariaDB:

```bash
sudo mariadb
```

Создайте базу и пользователя:

```sql
CREATE DATABASE sroki_godnosti CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'sroki'@'localhost' IDENTIFIED BY '8852285';
GRANT ALL PRIVILEGES ON sroki_godnosti.* TO 'sroki'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

Используется существующий пароль пользователя MariaDB: `8852285`.

## Установка структуры БД

Выполните SQL-скрипт из проекта:

```bash
mysql -u sroki -p sroki_godnosti < database/install.sql
```

Скрипт создает таблицы:

### `batches`

Партии товаров:

- `id`
- `created_at`
- `article`
- `quantity`
- `expiry_date`
- `days_left`
- `status`
- `updated_at`

### `logs`

Журнал действий:

- `id`
- `action`
- `payload`
- `created_at`

### `settings`

Настройки уведомлений. Для совместимости в таблице остаются поля `notify_*`, но ежедневный cron-скрипт использует фиксированные критерии: срок истек, осталось 15, 30 или 60 дней. Получатели берутся из `notification_email`:

- `id`
- `notify_90_days`
- `notify_60_days`
- `notify_30_days`
- `notify_15_days`
- `notify_7_days`
- `notify_1_day`
- `notification_email`

## Настройка подключения к MariaDB

Основной файл подключения:

```text
app/database.php
```

По умолчанию используются параметры:

```php
$host = 'localhost';
$database = 'sroki_godnosti';
$user = 'sroki';
$password = '8852285';
```

На VPS рекомендуется не хранить пароль в репозитории, а задать переменные окружения веб-сервера и cron-пользователя:

```bash
export DB_HOST=localhost
export DB_NAME=sroki_godnosti
export DB_USER=sroki
export DB_PASSWORD='8852285'
export DB_CHARSET=utf8mb4
```

Если переменные окружения недоступны, измените значения по умолчанию в `app/database.php` на сервере.

## API

API находится в `public/api.php`.

### Получение списка партий

```http
GET /api.php?action=list
```

Поддерживаемые параметры поиска и фильтрации:

- `article` — поиск по артикулу;
- `search` — общий поиск по артикулу;
- `status` — фильтр по статусу;
- `days_from` и `days_to` — фильтр по остатку дней;
- `date_from` и `date_to` — фильтр по периоду срока годности.

### Добавление партии

```http
POST /api.php?action=create
Content-Type: application/json
```

Пример тела:

```json
{
  "article": "12345",
  "quantity": 10,
  "expiryDate": "2026-07-01",
  "status": "В наличии"
}
```

### Массовый импорт партий

```http
POST /api.php?action=bulk_create
Content-Type: application/json
```

```json
{
  "batches": [
    {
      "article": "12345",
      "quantity": 10,
      "expiryDate": "2026-07-01"
    }
  ]
}
```

### Изменение партии

```http
POST /api.php?action=update
```

Тело должно содержать `id` и обновленные поля партии.

### Удаление партии

```http
POST /api.php?action=delete
```

```json
{ "id": 1 }
```

### Формирование отчетов

```http
GET /api.php?action=report&type=15
```

Поддерживаемые типы:

- `expired` — просроченные;
- `15` — до 15 дней;
- `30` — до 30 дней;
- `60` — до 60 дней;
- `custom&days_from=10&days_to=45` — произвольный диапазон.

В отчеты попадают только партии со статусом `В наличии`.

## Импорт XLSX

1. Откройте вкладку `Загрузка партии`.
2. При необходимости нажмите `Скачать шаблон таблицы`, чтобы получить XLSX с правильной структурой колонок.
3. Выберите XLSX-файл.
4. Обязательные колонки:
   - `Артикул`;
   - `Количество в партии`;
   - `Срок годности до`.
5. Необязательные колонки:
   - `Дата внесения`;
   - `Статус партии`.
6. Нажмите `Загрузить строки`.

## Экспорт XLSX

- Во вкладке `Реестр` нажмите:
  - `Выгрузить фильтр XLSX` для текущего результата фильтрации;
  - `Выгрузить все XLSX` для полного реестра.

В XLSX-выгрузку попадают только партии со статусом `В наличии`; партии `Реализована` и `Списана` исключаются.

## Настройка Cron для ежедневной проверки

Скрипт проверки:

```text
scripts/check_expiry.php
```

Добавьте cron-задачу для ежедневного запуска в 09:00:

```bash
crontab -e
```

```cron
0 9 * * * /usr/bin/php /var/www/kvasmix.ru/vr/sroki_godnosti/scripts/check_expiry.php >> /var/log/sroki_godnosti_expiry.log 2>&1
```

Скрипт:

- пересчитывает `days_left`;
- выбирает партии со статусом `В наличии`;
- проверяет фиксированные критерии: срок истек, осталось 15, 30 или 60 дней;
- отправляет письмо всем получателям из настройки `notification_email`;
- пишет результат в таблицу `logs`.

Письмо содержит:

- артикул;
- количество;
- остаток дней;
- срок годности;
- ссылку на реестр с фильтром по артикулу.

## Настройка email-уведомлений

Уведомления отправляет cron-скрипт `scripts/check_expiry.php`. Получатели берутся из вкладки `Настройки`, а письмо отправляется от имени `vr-vk@yandex.ru`.

Сервис каждый день в 09:00 по МСК проверяет партии со статусом `В наличии` и отправляет письмо, если есть хотя бы одна партия по критериям:

- срок годности партии истек;
- срок годности заканчивается через 15 дней;
- срок годности заканчивается через 30 дней;
- срок годности заканчивается через 60 дней.

Если под критерии попали несколько партий, они перечисляются в одном письме. Если подходящих партий нет, письмо не отправляется.

Текст по каждой партии содержит:

- тип уведомления;
- артикул;
- количество;
- срок годности;
- просьбу актуализировать статус товара в реестре;
- ссылку на реестр с примененным фильтром по текущему артикулу, например `https://kvasmix.ru/vr/sroki_godnosti/?tab=registry&article=12345`.

### 1. Добавьте получателей в интерфейсе

1. Откройте админ-панель `https://kvasmix.ru/vr/sroki_godnosti/`.
2. Перейдите во вкладку `Настройки`.
3. В блоке `Email получатели` введите адрес, например `manager@example.com`.
4. Нажмите `Добавить`.
5. Повторите действие для всех получателей.

После сохранения адреса попадают в поле `settings.notification_email` в MariaDB. Несколько адресов хранятся через запятую. Уведомления отправляются всем добавленным получателям.

### 2. Настройте связь сайта с Яндекс Почтой `vr-vk@yandex.ru`

Рекомендуемый вариант — отправка через SMTP Яндекс Почты по паролю приложения.

1. Войдите в аккаунт `vr-vk@yandex.ru`.
2. Откройте настройки безопасности Яндекс ID.
3. Включите двухфакторную защиту, если Яндекс требует это для паролей приложений.
4. Создайте пароль приложения для почты / SMTP.
5. Сохраните пароль приложения: он показывается один раз.

На VPS задайте переменные окружения для cron-пользователя:

```bash
export SMTP_HOST=smtp.yandex.ru
export SMTP_PORT=465
export SMTP_USERNAME=vr-vk@yandex.ru
export SMTP_PASSWORD='ПАРОЛЬ_ПРИЛОЖЕНИЯ_ЯНДЕКС'
export SMTP_FROM_NAME='Сроки годности'
export APP_URL='https://kvasmix.ru/vr/sroki_godnosti/'
```

Если переменные окружения удобнее хранить в отдельном файле, создайте, например, `/var/www/kvasmix.ru/vr/sroki_godnosti/.env.cron`:

```bash
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USERNAME=vr-vk@yandex.ru
SMTP_PASSWORD='ПАРОЛЬ_ПРИЛОЖЕНИЯ_ЯНДЕКС'
SMTP_FROM_NAME='Сроки годности'
APP_URL='https://kvasmix.ru/vr/sroki_godnosti/'
```

И подключите его в cron перед запуском скрипта:

```cron
0 9 * * * set -a; . /var/www/kvasmix.ru/vr/sroki_godnosti/.env.cron; set +a; /usr/bin/php /var/www/kvasmix.ru/vr/sroki_godnosti/scripts/check_expiry.php >> /var/log/sroki_godnosti_expiry.log 2>&1
```

Если `SMTP_PASSWORD` не задан, скрипт попробует отправить письмо через стандартную PHP-функцию `mail()` с заголовком `From: vr-vk@yandex.ru`, но для стабильной доставки лучше использовать SMTP Яндекса.

### 3. Проверьте SMTP-доступ с VPS

Проверьте, что сервер видит SMTP Яндекса:

```bash
openssl s_client -connect smtp.yandex.ru:465 -crlf
```

В ответе должен быть сертификат и строка SMTP-сервера. Для выхода введите `QUIT`.

### 4. Проверьте cron-скрипт вручную

Запустите проверку сроков из каталога проекта:

```bash
set -a; . /var/www/kvasmix.ru/vr/sroki_godnosti/.env.cron; set +a
/usr/bin/php /var/www/kvasmix.ru/vr/sroki_godnosti/scripts/check_expiry.php
```

Скрипт отправит письмо только если одновременно выполнены условия:

- в настройках есть хотя бы один email-получатель;
- в таблице `batches` есть партии со статусом `В наличии`;
- `days_left` партии меньше 0 либо равен 15, 30 или 60.

Результат запуска записывается в таблицу `logs`. Если писем нет, проверьте, есть ли в реестре партии, которые сегодня попадают под критерии уведомлений.

### 5. Проверьте ежедневное расписание

Cron-задача должна запускать проверку каждый день в 09:00 по московскому времени. Если системное время сервера не МСК, укажите `TZ=Europe/Moscow` в crontab:

```cron
TZ=Europe/Moscow
0 9 * * * set -a; . /var/www/kvasmix.ru/vr/sroki_godnosti/.env.cron; set +a; /usr/bin/php /var/www/kvasmix.ru/vr/sroki_godnosti/scripts/check_expiry.php >> /var/log/sroki_godnosti_expiry.log 2>&1
```

Проверить установленные cron-задачи можно командой:

```bash
crontab -l
```

Лог запуска cron смотрите в файле:

```bash
tail -n 100 /var/log/sroki_godnosti_expiry.log
```

### 6. Если письма не приходят

Проверьте:

- добавлены ли получатели во вкладке `Настройки`;
- есть ли партии со статусом `В наличии` и остатком дней `истек`, `15`, `30` или `60`;
- задан ли `SMTP_PASSWORD` именно как пароль приложения Яндекса, а не обычный пароль от аккаунта;
- открыт ли исходящий доступ с VPS к `smtp.yandex.ru:465`;
- не попадает ли письмо в спам.

Для лучшей доставляемости проверьте DNS-записи домена и почтового ящика: SPF, DKIM и DMARC.

## Работа со статусами

Статусы партий:

- `В наличии` — партия участвует в отчетах и уведомлениях;
- `Реализована` — партия остается в базе, но исключается из отчетов и уведомлений;
- `Списана` — партия остается в базе, но исключается из отчетов и уведомлений.

Партии не удаляются автоматически при изменении статуса. Они остаются в таблице `batches`, что позволяет хранить историю и выгружать полный реестр.

## История

Таблица `logs` хранит действия API и cron-скрипта. В интерфейсе вкладка `История` показывает изменения в реестре: добавление, импорт, обновление и удаление партий.

## Безопасность

- Не размещайте `app/`, `database/` и `scripts/` как публичные каталоги.
- Используйте сильный пароль пользователя MariaDB.
- Ограничьте доступ к URL админ-панели средствами веб-сервера, если сервисом должны пользоваться только сотрудники.
- Регулярно проверяйте резервные копии и возможность восстановления.
