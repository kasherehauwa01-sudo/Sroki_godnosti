# Сервис контроля сроков годности партий товаров

Веб-сервис предназначен для учета партий товаров, контроля сроков годности, формирования отчетов и автоматической отправки уведомлений. Интерфейс рассчитан на размещение на сервере Timeweb по адресу `https://kvasmix.ru/vr/sroki_godnosti/`, а данные хранятся в Google Таблице:

`https://docs.google.com/spreadsheets/d/1-T4jxUkLZf1zezuVfDNNoq288jJD6jnWtjAMhzo3xg4/edit?gid=0#gid=0`

## Что реализовано

- Административная панель с вкладками:
  - загрузка новой партии;
  - отчеты по срокам годности;
  - реестр партий;
  - настройки email-получателей и правил уведомлений;
  - логи сервиса.
- Ручное добавление партий с полями: артикул, наименование, количество, срок годности.
- Импорт XLSX в браузере через SheetJS. В файле должны быть колонки: `Артикул`, `Наименование`, `Количество в партии`, `Срок годности до`.
- Выгрузка в XLSX:
  - отчетов;
  - полного реестра;
  - результатов фильтрации.
- Быстрый поиск и фильтрация без перезагрузки страницы.
- Цветовая индикация сроков годности:
  - без индикации — более 60 дней;
  - желтый — до 60 дней;
  - оранжевый — до 30 дней;
  - красный — до 15 дней;
  - серый — просрочено.
- Изменение статуса партии: `В наличии`, `Реализована`, `Списана`.
- Перенос реализованных и списанных партий в лист `Архив`.
- Google Apps Script для приема данных, чтения реестра, ежедневных уведомлений и резервного копирования.
- Ежедневная проверка сроков годности в 09:00.
- Ежемесячная резервная копия XLSX на `vr-vk@yandex.ru` каждого 1 числа месяца в 09:00.

## Требования

### На Timeweb

- PHP 8.0 или новее.
- Расширение `curl` для PHP.
- Доступ сайта по HTTPS.
- Возможность загрузить файлы в каталог сайта, например в `/vr/sroki_godnosti/`.

### В Google

- Google Таблица с ID `1-T4jxUkLZf1zezuVfDNNoq288jJD6jnWtjAMhzo3xg4`.
- Google Apps Script, опубликованный как Web App.
- Разрешения Apps Script на работу с Google Sheets, MailApp, UrlFetchApp и триггерами.

## Установка проекта на сервер Timeweb

1. Скопируйте файлы проекта на сервер.
2. Для URL `https://kvasmix.ru/vr/sroki_godnosti/` содержимое каталога `public/` должно быть доступно как корень этого URL.
3. Каталог `app/` лучше разместить выше публичного каталога. Если структура хостинга не позволяет, закройте прямой доступ к `app/` правилами веб-сервера.
4. Скопируйте файл настроек:

```bash
cp app/config.example.php app/config.php
```

5. Откройте `app/config.php` и заполните:

```php
return [
    'gas_web_app_url' => 'https://script.google.com/macros/s/ВАШ_DEPLOYMENT_ID/exec',
    'api_secret' => 'длинная-случайная-строка',
    'request_timeout' => 30,
];
```

6. Убедитесь, что файл `public/api.php` может прочитать `../app/config.php`. Если на сервере используется другая структура каталогов, скорректируйте путь `$configPath` в `public/api.php`.

## Настройка окружения

Рекомендуемые права доступа:

- `public/index.php`, `public/api.php`, `public/assets/*` — чтение для веб-сервера.
- `app/config.php` — чтение для веб-сервера, без публичного доступа из браузера.

Проверка PHP на сервере:

```bash
php -v
php -m | grep curl
```

Если расширение `curl` отключено, включите его в панели управления Timeweb или обратитесь в поддержку хостинга.

## Настройка Google Таблицы

Apps Script автоматически создаст листы при первом запросе или при ручном запуске `ensureStructure()`:

- `Партии` — активные партии со статусом `В наличии`.
- `Архив` — партии со статусами `Реализована` и `Списана`.
- `Настройки` — JSON с получателями и правилами уведомлений.
- `Логи` — события работы сервиса.

Колонки листа `Партии` и `Архив`:

| Колонка | Назначение |
| --- | --- |
| `id` | Технический идентификатор партии |
| `Дата внесения` | Дата добавления партии |
| `Артикул` | Артикул товара |
| `Наименование` | Наименование товара |
| `Количество в партии` | Количество |
| `Срок годности до` | Дата окончания срока годности |
| `Статус партии` | `В наличии`, `Реализована`, `Списана` |
| `Дата архивации` | Дата переноса в архив |

## Настройка Google Apps Script

1. Откройте Google Таблицу.
2. Перейдите в `Расширения` → `Apps Script`.
3. Вставьте содержимое файла `scripts/google-apps-script.gs`.
4. Откройте `Project Settings` → `Script Properties`.
5. Добавьте свойство:

| Key | Value |
| --- | --- |
| `API_SECRET` | такая же длинная случайная строка, как в `app/config.php` |

6. Запустите функцию `ensureStructure()` вручную и выдайте разрешения.
7. Запустите функцию `installTriggers()` вручную и выдайте разрешения.
8. Опубликуйте скрипт: `Deploy` → `New deployment` → тип `Web app`.
9. Рекомендуемые параметры публикации:
   - `Execute as`: `Me`.
   - `Who has access`: вариант, позволяющий вашему серверу отправлять запросы к Web App. Обычно используется `Anyone`, так как доступ дополнительно защищен `API_SECRET`.
10. Скопируйте URL вида `https://script.google.com/macros/s/.../exec` в `app/config.php`.

## Настройка почтовых уведомлений

1. Откройте веб-интерфейс сервиса.
2. Перейдите на вкладку `Настройки`.
3. Добавьте email-получателей уведомлений.
4. Добавьте правила уведомлений. Примеры:
   - `15` — уведомлять о партиях, срок которых истекает ровно через 15 дней;
   - `30` — уведомлять о партиях, срок которых истекает ровно через 30 дней;
   - `60` — уведомлять о партиях, срок которых истекает ровно через 60 дней;
   - `-1` — уведомлять о просроченных партиях.
5. Apps Script отправляет письма функцией `MailApp.sendEmail` от имени Google-аккаунта владельца скрипта.

Рассылка запускается триггером `dailyExpiryCheck` ежедневно в 09:00. Письмо отправляется только если найдены подходящие партии со статусом `В наличии`.

Пример текста уведомления:

```text
У партии товара арт. 12345 истекает срок годности через 15 дней. Наименование: Квас. Количество: 20.
```

## Импорт данных

1. Подготовьте XLSX-файл с первой строкой заголовков.
2. Обязательные колонки:
   - `Артикул`;
   - `Наименование`;
   - `Количество в партии`;
   - `Срок годности до`.
3. Откройте вкладку `Загрузка партии`.
4. Выберите XLSX-файл.
5. Проверьте количество найденных строк в блоке предпросмотра.
6. Нажмите `Загрузить строки`.

Дата внесения проставляется автоматически, если ее нет в импортируемых данных. Статус по умолчанию — `В наличии`.

## Формирование отчетов

1. Откройте вкладку `Отчеты`.
2. Выберите категорию:
   - `Просроченные партии`;
   - `Истекают через 15 дней`;
   - `Истекают через 30 дней`;
   - `Истекают через 60 дней`;
   - `Пользовательский период`.
3. Для пользовательского периода укажите диапазон дней.
4. Нажмите `Сформировать`.
5. Для выгрузки нажмите `Выгрузить XLSX`.

В отчеты попадают только партии со статусом `В наличии`. Партии со статусами `Реализована` и `Списана` не участвуют в отчетах и уведомлениях.

## Работа со статусами партий

1. Откройте вкладку `Реестр`.
2. Найдите партию через поиск или фильтры.
3. Измените значение в колонке `Статус`.
4. Если выбран статус `Реализована` или `Списана`, партия переносится из листа `Партии` в лист `Архив`.
5. Архивные партии остаются доступными в общем реестре, но не участвуют в уведомлениях и отчетах.

## Выгрузка данных

На вкладке `Реестр` доступны две выгрузки:

- `Выгрузить фильтр XLSX` — экспортирует текущий результат поиска и фильтрации.
- `Выгрузить все XLSX` — экспортирует полный список партий, включая архивные.

На вкладке `Отчеты` кнопка `Выгрузить XLSX` экспортирует текущий отчет.

## Резервное копирование данных

Функция `monthlyBackup()` создает XLSX-экспорт Google Таблицы и отправляет его на `vr-vk@yandex.ru` каждого 1 числа месяца в 09:00. Тема письма формируется в формате:

```text
Сроки годности. Резервная копия от дд.мм.гггг
```

Для включения расписания необходимо один раз вручную запустить `installTriggers()` в Apps Script.

## Логи

Вкладка `Логи` показывает последние события из листа `Логи`, включая:

- добавление партий;
- изменение статусов;
- перенос в архив;
- сохранение настроек;
- ежедневные проверки;
- отправку уведомлений;
- ошибки API;
- резервное копирование.

## Проверка после установки

1. Откройте `https://kvasmix.ru/vr/sroki_godnosti/`.
2. Если отображается ошибка настройки, проверьте `app/config.php`.
3. Добавьте тестовую партию со сроком годности через 15 дней.
4. Убедитесь, что партия появилась в реестре и Google Таблице.
5. Сформируйте отчет `Истекают через 15 дней`.
6. Добавьте email в настройках.
7. В Apps Script вручную запустите `dailyExpiryCheck()` и проверьте получение письма.
8. В Apps Script вручную запустите `monthlyBackup()` и проверьте получение XLSX-копии.

## Безопасность

- Не публикуйте `app/config.php` в Git.
- Используйте длинный случайный `API_SECRET`.
- Регулярно проверяйте лист `Логи`.
- Ограничьте доступ к административному URL средствами Timeweb, если панель должна быть доступна только сотрудникам.
