# Контроль сроков годности партий товаров

Веб-сервис для учета партий товаров, контроля сроков годности, формирования отчетов, импорта/экспорта XLSX и ежедневных email-уведомлений. Проект работает по адресу:

```text
https://kvasmix.ru/vr/sroki_godnosti/
```

Новая архитектура проекта:

```text
Сайт → PHP API → MariaDB
```

## Возможности

- Вкладка `Загрузка партии`:
  - ручное добавление партии;
  - импорт XLSX с колонками `Артикул`, `Наименование`, `Количество в партии`, `Срок годности до`;
  - дополнительно поддерживается колонка `Код`.
- Вкладка `Отчеты`:
  - просроченные партии;
  - партии до 15, 30 и 60 дней;
  - произвольный диапазон;
  - выгрузка отчета в XLSX.
- Вкладка `Реестр`:
  - список всех партий;
  - поиск по артикулу, коду и наименованию;
  - фильтры по статусу, остатку дней и периоду срока годности;
  - изменение статуса партии;
  - выгрузка полного списка или результата фильтрации в XLSX.
- Вкладка `Настройки`:
  - добавление и удаление email-получателей;
  - включение уведомлений за 90, 60, 30, 15, 7 или 1 день.
- Вкладка `Логи`:
  - журнал создания, обновления, удаления, изменения настроек и cron-проверок.
- Цветовая индикация в реестре:
  - без индикации — более 60 дней;
  - желтый — до 60 дней;
  - оранжевый — до 30 дней;
  - красный — до 15 дней;
  - серый — просрочено.

## Требования к серверу Timeweb VPS

- Linux VPS с доступом по SSH.
- Nginx или Apache.
- PHP 8.0 или новее.
- PHP-расширения: `pdo_mysql`, `mbstring`, `json`.
- MariaDB 10.5+ или MySQL 8+.
- Настроенная отправка почты с сервера для функции PHP `mail()` или внешний MTA.
- Cron для ежедневного запуска проверки сроков.

## Установка MariaDB

Пример для Ubuntu/Debian:

```bash
sudo apt update
sudo apt install mariadb-server mariadb-client
sudo systemctl enable --now mariadb
sudo mysql_secure_installation
```

Проверьте, что MariaDB запущена:

```bash
systemctl status mariadb
```

## Создание базы данных и пользователя

Подключитесь к MariaDB:

```bash
sudo mariadb
```

Создайте базу и пользователя:

```sql
CREATE DATABASE sroki_godnosti CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'sroki'@'localhost' IDENTIFIED BY '8852285';
GRANT ALL PRIVILEGES ON sroki_godnosti.* TO 'sroki'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

Используется существующий пароль пользователя MariaDB: `8852285`.

## Установка структуры БД

Выполните SQL-скрипт из проекта:

```bash
mysql -u sroki -p sroki_godnosti < database/install.sql
```

Скрипт создает таблицы:

### `batches`

Партии товаров:

- `id`
- `created_at`
- `article`
- `code`
- `name`
- `quantity`
- `expiry_date`
- `days_left`
- `status`
- `updated_at`

### `logs`

Журнал действий:

- `id`
- `action`
- `payload`
- `created_at`

### `settings`

Настройки уведомлений:

- `id`
- `notify_90_days`
- `notify_60_days`
- `notify_30_days`
- `notify_15_days`
- `notify_7_days`
- `notify_1_day`
- `notification_email`

## Настройка подключения к MariaDB

Основной файл подключения:

```text
app/database.php
```

По умолчанию используются параметры:

```php
$host = 'localhost';
$database = 'sroki_godnosti';
$user = 'sroki';
$password = '8852285';
```

На VPS рекомендуется не хранить пароль в репозитории, а задать переменные окружения веб-сервера и cron-пользователя:

```bash
export DB_HOST=localhost
export DB_NAME=sroki_godnosti
export DB_USER=sroki
export DB_PASSWORD='8852285'
export DB_CHARSET=utf8mb4
```

Если переменные окружения недоступны, измените значения по умолчанию в `app/database.php` на сервере.

## API

API находится в `public/api.php`.

### Получение списка партий

```http
GET /api.php?action=list
```

Поддерживаемые параметры поиска и фильтрации:

- `article` — поиск по артикулу;
- `code` — поиск по коду;
- `name` — поиск по наименованию;
- `search` — общий поиск по артикулу, коду и наименованию;
- `status` — фильтр по статусу;
- `days_from` и `days_to` — фильтр по остатку дней;
- `date_from` и `date_to` — фильтр по периоду срока годности.

### Добавление партии

```http
POST /api.php?action=create
Content-Type: application/json
```

Пример тела:

```json
{
  "article": "12345",
  "code": "ABC-001",
  "name": "Товар",
  "quantity": 10,
  "expiryDate": "2026-07-01",
  "status": "В наличии"
}
```

### Массовый импорт партий

```http
POST /api.php?action=bulk_create
Content-Type: application/json
```

```json
{
  "batches": [
    {
      "article": "12345",
      "name": "Товар",
      "quantity": 10,
      "expiryDate": "2026-07-01"
    }
  ]
}
```

### Изменение партии

```http
POST /api.php?action=update
```

Тело должно содержать `id` и обновленные поля партии.

### Удаление партии

```http
POST /api.php?action=delete
```

```json
{ "id": 1 }
```

### Формирование отчетов

```http
GET /api.php?action=report&type=15
```

Поддерживаемые типы:

- `expired` — просроченные;
- `15` — до 15 дней;
- `30` — до 30 дней;
- `60` — до 60 дней;
- `custom&days_from=10&days_to=45` — произвольный диапазон.

В отчеты попадают только партии со статусом `В наличии`.

## Импорт XLSX

1. Откройте вкладку `Загрузка партии`.
2. При необходимости нажмите `Скачать шаблон таблицы`, чтобы получить XLSX с правильной структурой колонок.
3. Выберите XLSX-файл.
4. Обязательные колонки:
   - `Артикул`;
   - `Наименование`;
   - `Количество в партии`;
   - `Срок годности до`.
5. Необязательные колонки:
   - `Код`;
   - `Дата внесения`;
   - `Статус партии`.
6. Нажмите `Загрузить строки`.

## Экспорт XLSX

- Во вкладке `Отчеты` нажмите `Выгрузить XLSX`.
- Во вкладке `Реестр` нажмите:
  - `Выгрузить фильтр XLSX` для текущего результата фильтрации;
  - `Выгрузить все XLSX` для полного реестра.

## Настройка Cron для ежедневной проверки

Скрипт проверки:

```text
scripts/check_expiry.php
```

Добавьте cron-задачу для ежедневного запуска в 09:00:

```bash
crontab -e
```

```cron
0 9 * * * /usr/bin/php /var/www/kvasmix.ru/vr/sroki_godnosti/scripts/check_expiry.php >> /var/log/sroki_godnosti_expiry.log 2>&1
```

Скрипт:

- пересчитывает `days_left`;
- выбирает партии со статусом `В наличии`;
- проверяет включенные правила уведомлений;
- отправляет письмо получателям из настройки `notification_email`;
- пишет результат в таблицу `logs`.

Письмо содержит:

- артикул;
- код, если заполнен;
- наименование;
- количество;
- остаток дней;
- срок годности;

## Настройка email-уведомлений

1. Откройте вкладку `Настройки`.
2. Добавьте один или несколько email-адресов.
3. Включите нужные правила: 90, 60, 30, 15, 7 или 1 день.
4. Убедитесь, что сервер умеет отправлять почту через PHP `mail()`.

Проверка отправки почты:

```bash
php -r "mail('you@example.com','Проверка','Проверка отправки почты');"
```

Если письма не отправляются, настройте Postfix/Exim или SMTP-релей на VPS.

## Резервное копирование

### Резервная копия базы

Создайте каталог для бэкапов:

```bash
sudo mkdir -p /var/backups/sroki_godnosti
sudo chown $USER:$USER /var/backups/sroki_godnosti
```

Команда ручного бэкапа:

```bash
mysqldump -u sroki -p sroki_godnosti > /var/backups/sroki_godnosti/sroki_godnosti_$(date +%F).sql
```

Cron для ежедневного бэкапа в 02:30:

```cron
30 2 * * * mysqldump -u sroki -p'8852285' sroki_godnosti | gzip > /var/backups/sroki_godnosti/sroki_godnosti_$(date +\%F).sql.gz
```

### Резервная копия файлов проекта

```bash
tar -czf /var/backups/sroki_godnosti/project_$(date +%F).tar.gz /var/www/kvasmix.ru/vr/sroki_godnosti
```

## Работа со статусами

Статусы партий:

- `В наличии` — партия участвует в отчетах и уведомлениях;
- `Реализована` — партия остается в базе, но исключается из отчетов и уведомлений;
- `Списана` — партия остается в базе, но исключается из отчетов и уведомлений.

Партии не удаляются автоматически при изменении статуса. Они остаются в таблице `batches`, что позволяет хранить историю и выгружать полный реестр.

## Логи

Таблица `logs` хранит действия API и cron-скрипта. В интерфейсе отображаются последние 300 записей.

## Безопасность

- Не размещайте `app/`, `database/` и `scripts/` как публичные каталоги.
- Используйте сильный пароль пользователя MariaDB.
- Ограничьте доступ к URL админ-панели средствами веб-сервера, если сервисом должны пользоваться только сотрудники.
- Регулярно проверяйте резервные копии и возможность восстановления.
