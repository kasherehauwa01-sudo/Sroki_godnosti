# Контроль сроков годности партий товаров

Веб-сервис для учета партий товаров, контроля сроков годности, работы со складами, сбора остатков, автоматических загрузок из почты и email-уведомлений. Проект работает по адресу:

```text
https://kvasmix.ru/vr/sroki_godnosti/
```

Архитектура проекта:

```text
Веб-интерфейс → PHP API → MariaDB
Cron/автозагрузка → PHP API/общие функции → MariaDB
```

## Основные возможности сервиса

### Реестр партий

- Хранение партий товаров с артикулом, кодом, наименованием, сроком годности, датой внесения, источником создания и статусом.
- Поддержка статусов `В наличии`, `Реализована`, `Списана`, `Нет в наличии`.
- Поиск по артикулу, коду или наименованию.
- Фильтры по статусу, остатку дней до срока годности и событию по сроку годности.
- Сортировка по артикулу, сроку годности, остатку дней и дате внесения.
- Ручное добавление одной или нескольких партий.
- Импорт партий из XLS/XLSX, включая старые XLS-файлы с русскими заголовками в Windows-1251.
- Редактирование партии.
- Защищенная паролем смена статуса, одиночное удаление и массовое удаление выбранных строк.
- Экспорт отфильтрованного результата в XLSX.
- Цветовая индикация сроков годности:
  - без индикации — более 60 дней;
  - желтый — до 60 дней;
  - оранжевый — до 30 дней;
  - красный — до 15 дней;
  - серый — срок годности уже истек.

### События по срокам годности

- В сервисе используются 5 типов событий по срокам годности:
  - 180 дней;
  - 90 дней;
  - 60 дней;
  - 30 дней;
  - 1 день.
- Вкладка `События` показывает прошедшие, сегодняшние и будущие события.
- По умолчанию отображаются сегодняшние и будущие события.
- События группируются по типу события и дате события.
- В строке события отображаются:
  - тип события;
  - дата события;
  - количество партий в событии.
- При клике на строку события открывается окно со списком партий события: `Артикул`, `Код`, `Наименование`.

### Уведомления и занесение остатков

- Сервис отправляет складские уведомления по событиям сроков годности.
- Для каждого активного склада формируется ссылка на форму занесения остатков.
- Склад переходит по ссылке и заполняет остатки по партиям.
- Остатки сохраняются по связке `партия + склад`.
- Если склад повторно открывает ссылку в период действия формы, он видит предыдущие значения и может их изменить.
- Вкладка `Уведомления` показывает партии, по которым склады внесли остатки.
- В уведомлениях отображается общий остаток по партии и колонка `Заполнили остатки` в формате `x из y`, где:
  - `x` — количество активных складов, по которым остатки уже заполнены;
  - `y` — текущее количество активных складов.
- Если склад добавлен, удален или деактивирован, значение `y` пересчитывается по текущему списку активных складов.
- Партии со статусом `Списана` не попадают в формы заполнения остатков и в список складских уведомлений.
- Из вкладки `Уведомления` можно открыть партию и посмотреть остатки по активным складам.
- В окне остатков доступна кнопка `Списать`: после нажатия появляется выбор статуса, пользователь выбирает статус и нажимает `Сохранить`. Статус партии обновляется в реестре.

### Склады

- Управление складами находится во вкладке `Настройки` → `Склады`.
- Для склада задаются:
  - название;
  - порядок отображения;
  - email-адреса для уведомлений;
  - активность.
- В рассылку и расчеты заполнения остатков попадают только активные склады.
- Главная вкладка `Склады` не используется: работа со складами выполняется из настроек.

### Рассылка товаров без фильтров

- Сервис умеет анализировать файл ежедневной выгрузки и находить товары без фильтра `Срок годности`.
- Для таких товаров формируется отдельная email-рассылка ответственным получателям.
- Получатели рассылки настраиваются во вкладке `Настройки` в блоке уведомлений `Товар без фильтров`.
- В настройках доступна тестовая отправка и просмотр логов рассылки товаров без фильтров.
- Рассылка помогает обнаружить товары, которые не попадут в контроль сроков годности из-за отсутствующего фильтра в исходной выгрузке.

### Автозагрузка партий из почты

- Сервис может автоматически забирать ежедневную XLS/XLSX-выгрузку из почтового ящика.
- Ожидаемые параметры письма:
  - отправитель: `robot_volgorost@volgorost.ru`;
  - тема: `Сроки годности. Ежедневная выгрузка`.
- Автозагрузка запускается в 23:50 по московскому времени: до 00:00 ищет непрочитанное письмо за текущий день, после 00:00 ищет непрочитанное письмо и за дату предыдущего запуска, и за новые сутки.
- Если письмо не найдено, сервис повторяет поиск каждые 30 минут, максимум 20 попыток.
- После успешной загрузки письмо помечается прочитанным, чтобы не загрузить его повторно.
- Результат автозагрузки сохраняется в истории: количество добавленных партий, дубли, списанные замещенные партии и ошибки.
- В настройках доступен ручной тест автозагрузки и просмотр логов автозагрузки.

### Автоматическое списание партий с кодом `-1`

- При добавлении партии с кодом, который заканчивается на `-1`, сервис считает ее заменяющей партией.
- Пример: если добавляется партия с кодом `XXXXX-1`, сервис ищет ранее внесенные партии с кодом `XXXXX`.
- Автоматическое списание выполняется только если у сравниваемых партий одинаковый срок годности.
- Найденные партии с базовым кодом переводятся в статус `Списана`.
- Уже списанные партии повторно не обрабатываются.
- Правило работает при:
  - ручном добавлении партий;
  - массовом добавлении партий;
  - импорте XLS/XLSX через интерфейс;
  - автозагрузке из почты.
- Информация об автоматически списанных партиях сохраняется в истории и в результате автозагрузки.

### История и аудит

- Вкладка `История` фиксирует ключевые действия:
  - ручное добавление партий;
  - массовое добавление;
  - автозагрузку;
  - редактирование;
  - смену статусов;
  - удаление;
  - отправку уведомлений;
  - тестовые действия;
  - ошибки автозагрузки и рассылок.
- В деталях истории сохраняется понятная информация о партии до и после изменения.

### Настройки

- Вкладка `Настройки` защищена паролем.
- В настройках доступны:
  - параметры дней уведомлений;
  - получатели основных уведомлений;
  - SMTP-параметры;
  - история уведомлений;
  - получатели рассылки товаров без фильтров;
  - тест рассылки товаров без фильтров;
  - тест автозагрузки;
  - логи автозагрузки;
  - управление складами;
  - настройки заполнения остатков;
  - системная информация.

## Инструкция для пользователей

### 1. Поиск и фильтрация партий

1. Откройте вкладку `Реестр`.
2. Введите текст в поле поиска.
3. Выберите, где искать: `Артикул`, `Код` или `Наименование`.
4. При необходимости выберите фильтр по статусу, остатку дней или событию.
5. Чтобы вернуться к полному списку, нажмите `Сбросить фильтры`.

### 2. Ручное добавление партий

1. Нажмите `Добавить партию`.
2. Заполните обязательные поля.
3. Срок годности вводится в формате `мм.гггг` или `дд.мм.гггг`, например `08.2026` или `15.08.2026`.
4. Если нужно добавить несколько партий, нажмите `Добавить строку`.
5. Нажмите `Добавить партии в реестр`.

Если партия с таким же артикулом и сроком годности уже есть в реестре, система пропустит дубль и покажет сообщение.

### 3. Импорт XLS/XLSX

1. Нажмите `Добавить партию`.
2. В открывшемся окне нажмите `Загрузить XLS`.
3. При необходимости скачайте шаблон кнопкой `Скачать шаблон таблицы`.
4. Выберите файл `.xls` или `.xlsx`.
5. Проверьте предварительный результат.
6. Нажмите `Загрузить шаблон`.

Обязательные колонки файла:

- `Артикул`;
- `Срок годности до`, `Срок годности` или похожий заголовок.

Необязательные колонки:

- `Код`;
- `Наименование`;
- `Дата внесения`;
- `Статус партии`.

### 4. Смена статуса и списание

1. Нажмите `Списать / Удалить`.
2. Введите пароль ответственного пользователя.
3. После успешного ввода пароля можно менять статусы в реестре.
4. Выберите новый статус: `В наличии`, `Реализована` или `Списана`.

Статус также можно изменить из окна остатков партии во вкладке `Уведомления`.

### 5. Работа с событиями

1. Откройте вкладку `События`.
2. Выберите фильтры `Сегодня`, `Прошедшие`, `Будущие`.
3. Найдите нужную строку события по типу и дате.
4. Нажмите на строку, чтобы открыть список партий события.

### 6. Заполнение остатков складами

1. Склад получает email со ссылкой на форму заполнения остатков.
2. Ответственный сотрудник открывает ссылку.
3. Вносит остатки по партиям.
4. Нажимает сохранение формы.
5. Администратор видит прогресс заполнения во вкладке `Уведомления`.

### 7. Управление складами

1. Откройте вкладку `Настройки`.
2. Введите пароль.
3. Перейдите в подкладку `Склады`.
4. Добавьте склад или измените существующий.
5. Укажите email-адреса, на которые склад должен получать уведомления.
6. Отключите склад, если он не должен участвовать в рассылках и расчете заполнения остатков.

### 8. Настройка рассылки товаров без фильтров

1. Откройте вкладку `Настройки`.
2. В блоке `Уведомления «Товар без фильтров»` укажите получателей.
3. Нажмите `Тест`, чтобы проверить отправку.
4. Используйте `Логи`, чтобы посмотреть результат предыдущих отправок.

### 9. Автозагрузка

1. Заполните SMTP/IMAP-доступы в настройках.
2. Убедитесь, что ежедневное письмо приходит от нужного отправителя и с нужной темой.
3. Проверьте работу кнопкой `Тест автозагрузки`.
4. Настройте cron для регулярного запуска.
5. Контролируйте результат в логах автозагрузки.

## Структура проекта

```text
app/
  auto_importer.php          автозагрузка XLS/XLSX из почты
  database.php               подключение к MariaDB через PDO
  mailer.php                 отправка email
  notification_templates.php шаблоны уведомлений
  warehouse_repository.php   склады, остатки и складские уведомления
  config.example.php         пример локальной конфигурации
database/
  install.sql                SQL-скрипт создания структуры БД
public/
  index.php                  веб-интерфейс администратора
  api.php                    API, работающее напрямую с MariaDB
  fill-stock.php             публичная форма заполнения остатков складами
  assets/
    app.js                   клиентская логика интерфейса
    styles.css               стили интерфейса
scripts/
  auto_import.php            cron-скрипт автозагрузки
  check_expiry.php           cron-скрипт проверки сроков и уведомлений
README.md                    описание сервиса и инструкция
```

## Требования к серверу

- Linux VPS с доступом по SSH.
- Nginx или Apache.
- PHP 8.0 или новее.
- PHP-расширения: `pdo_mysql`, `mbstring`, `json`, `imap`.
- MariaDB 10.5+ или MySQL 8+.
- Доступ к SMTP-серверу для отправки уведомлений.
- Доступ к IMAP-ящику для автозагрузки.
- Cron для регулярных задач.

## Настройка подключения к MariaDB

Основной файл подключения:

```text
app/database.php
```

Подключение берет параметры из переменных окружения или локального файла `app/config.php`, который не нужно добавлять в git.

Пример переменных окружения:

```bash
export DB_HOST=localhost
export DB_NAME=sroki_godnosti
export DB_USER=sroki
export DB_PASSWORD='YOUR_DB_PASSWORD'
export DB_CHARSET=utf8mb4
```

## Установка структуры БД

```bash
mysql -u sroki -p sroki_godnosti < database/install.sql
```

## Cron-задачи

### Проверка сроков годности

```cron
0 9 * * * /usr/bin/php /var/www/kvasmix.ru/vr/sroki_godnosti/scripts/check_expiry.php >> /var/log/sroki_godnosti_expiry.log 2>&1
```

### Автозагрузка из почты

```cron
50 23 * * * /usr/bin/php /var/www/kvasmix.ru/vr/sroki_godnosti/scripts/auto_import.php >> /var/log/sroki_godnosti_auto_import.log 2>&1
```

## API

API находится в `public/api.php`.

Основные действия:

- `GET /api.php?action=list` — список партий;
- `POST /api.php?action=create` — добавление партии;
- `POST /api.php?action=bulk_create` — массовое добавление партий;
- `POST /api.php?action=update` — изменение партии;
- `POST /api.php?action=delete` — удаление партии;
- `GET /api.php?action=events` — события по срокам годности;
- `GET /api.php?action=stock_batch_notifications` — уведомления по партиям с остатками;
- `GET /api.php?action=batch_stock&batch_id=ID` — остатки партии по складам;
- `GET /api.php?action=warehouses` — склады;
- `POST /api.php?action=warehouse_create` — создание склада;
- `POST /api.php?action=warehouse_update` — изменение склада;
- `POST /api.php?action=warehouse_delete` — удаление или деактивация склада;
- `POST /api.php?action=test_missing_filter_notification` — тест рассылки товаров без фильтров;
- `POST /api.php?action=test_stock_fill_notification` — тест уведомления на заполнение остатков;
- `POST /api.php?action=test_auto_import` — тест автозагрузки.

## Важные правила эксплуатации

- Не храните реальные пароли от MariaDB, SMTP и IMAP в git.
- Перед удалением или массовой сменой статусов убедитесь, что введен пароль ответственного пользователя.
- Партии со статусом `Списана` исключаются из складских уведомлений.
- Если код новой партии заканчивается на `-1`, проверьте срок годности: автоматическое списание базовой партии сработает только при совпадении срока годности.
- После добавления или отключения склада показатель `x из y` во вкладке `Уведомления` будет считаться по текущему списку активных складов.
